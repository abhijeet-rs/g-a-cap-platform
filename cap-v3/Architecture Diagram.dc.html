The CAP Cloud Tool replaces a fragmented, Excel-driven Client Approved Plan workflow with a validated, event-driven microservices platform. The architecture is designed so that approved plan data is entered once and propagates automatically to every downstream system of record.

Six-layer platform: requests enter the Edge layer, orchestrate through the Domain Layer hubs and Core Services, push via the Connector Layer to external systems. Dashed arrows are async EventBus messages.
Authentication uses SAML 2.0 SSO + enforced MFA at the enterprise IdP (Okta / Azure AD), exchanged for short-lived OAuth 2.0 JWTs. CAP data includes PHI and PII — the platform is designed for HIPAA, SOC 2 Type II, and SOX compliance with layered trust zones and an immutable audit log.
