Core Concept: Dataforms — how ClientSpace stores all data
What is a Dataform?
A dynamic, configurable data collection page backed by a database table (prefixed gen_). Core entities — Organization, Contact, Client Master, Cases — are all Dataforms.
What is a Pipeline?
Business rules that fire when a Dataform is saved. Pipelines execute validation, trigger email templates, and — critically — expose fields to the API. Only API-enabled fields with a Pipeline are accessible via REST.
API Access Model
Dedicated API user accounts via System Admin > Advanced > API Users. Session-based auth (username/password) — not OAuth. Hierarchical security at Dataform and field level.
PrismHR ↔ ClientSpace Integration — dedicated bidirectional module (verified)
ClientSpace
Import (PrismHR → ClientSpace)
Nightly scheduled imports via Import Queue. ClientSpace evaluates each file’s PEOID against Secondary ID for workspace routing. Records marked as Import, Update, or Error.
Export (ClientSpace → PrismHR)
On-demand exports — Client Master must export first to create a PrismHR container, then Employees export into it. Strict sequential ordering enforced.

If APIs Are Unavailable — Fallback Approaches — if consultation items are unanswered or “no”
1
Salesforce Bridge
Preferred
ClientSpace already has a documented two-way Salesforce integration. Route our data through Salesforce as an intermediary — Salesforce → ClientSpace sync handles case creation, client records, and task management automatically.
Salesforce REST API ClientSpace auto-sync No direct CS API needed
2
Direct Database Integration
Reliable
Every Dataform creates a gen_ database table. With DB access, write directly to gen_Cases, gen_ClientMaster, etc. Pipelines fire on next UI access. Most reliable fallback — bypasses API entirely.
SQL write to gen_ tables Pipeline rules still execute
3
Power Automate / Zapier
Enterprise platforms without public REST APIs often have Power Automate connectors. Trigger flows from our system via webhook → Power Automate handles ClientSpace operations. Low-code, maintainable, auditable.
4
Headless Browser Automation
Last Resort
Playwright/Puppeteer automates the ClientSpace web UI — login, navigate to case form, fill fields, submit. Productionizable with error handling and retry logic. Works when no API, no DB access, and no connectors exist.
Risk Mitigation: These 4 approaches are ordered by preference. At least one will work regardless of ClientSpace’s API coverage — the Salesforce bridge and database integration are battle-tested patterns used across the PEO industry. No scenario leaves us without a viable integration path.
